Rule: react-doctor/no-pass-data-to-parent
Severity: warning
Category: Bugs
Count: 1

Handing data back to a parent from a useEffect costs your users an extra render.

Suggestion: Fetch the data in the parent and pass it down as a prop (or return it from the hook), instead of handing it back up through a prop callback in a useEffect. See https://react.dev/learn/you-might-not-need-an-effect#passing-data-to-the-parent

Curl with no cache & follow the canonical fix and false positive check recipe before fixing: https://react.doctor/docs/rules/react-doctor/no-pass-data-to-parent
