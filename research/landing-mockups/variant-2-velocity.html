Most platforms are slow because they're cautious. otterdeploy is fast because it's honest. The build runs locally, the cache lives forever, and the deploy is a function call — not a ceremony.

— Who it's for

Engineering teams that ship 40 times a day and don't want a control plane that's heavier than the app it's deploying. Designers, founders, and indie hackers who want production to feel like staging.

№02 — Inside the issue
Six chapters

An honest deployment platform,
covered chapter by chapter.

01
— Build · Chapter one

The build runs where you are.

BuildKit cached on disk, then mirrored to a remote farm of ARM and x86 builders. Every dependency you've ever pulled is one keystroke away. The build doesn't start when CI is ready — it starts when you are.

The result: a Next.js 15 monorepo with twelve packages ships in 11 seconds on warm cache. Cold, it takes 22. We have receipts.

Read the chapter →
02
— Run · Chapter two

Three regions, one mesh.

Internal services talk over a Wireguard mesh. External traffic terminates at Caddy. HTTP/3 by default. No load balancer to configure. No certificate to renew.

№03 — A deploy, in real time

What happens
in twenty-two seconds.

A real timeline of a real deploy. The same Next.js 15 app, the same git push, every step instrumented and captured. This is what happens between git push and your domain serving traffic.
0.0seconds

Webhook arrives from GitHub.

otterdeploy receives the push event, validates the signature, and queues the build. No CI runner to spin up — the build farm is already warm.

2.1seconds

Lockfile hit. Cache restored.

bun.lock hashes match the previous deploy. Node modules restored from layer cache in 1.4 seconds. Next.js .next folder restored too.

9.4seconds

Incremental build complete.

Only the 3 changed routes recompiled. TypeScript checked in parallel on a separate worker. No artifacts upload — the image streams to the registry as it builds.

14.8seconds

Image pushed. Replicas booted.

Three replicas come online across iad-1, fra-1, and sin-1. Each one gets a 200ms warm-up window before traffic shifts.

22.0shipped

Traffic shifted. The old replicas drain.

Health checks green across the mesh. Caddy updates its config in-memory — no reload, no dropped connections. Slack ping fires. You're back to coding.
