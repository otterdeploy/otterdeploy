# No Co-Author Commits

This skill enforces a strict repository policy: do not add co-author trailers or assistant attribution to commits or related git text.

Use this whenever work includes:

- creating a commit
- proposing a commit message
- preparing a squash merge message
- writing PR text that may be reused as commit history
- editing release/history text derived from commits

## Rules

- Never add a `Co-authored-by:` trailer.
- Never add assistant attribution such as `Generated by`, `Created with`, `Authored with`, `Claude`, `Codex`, `ChatGPT`, or `OpenAI` in commit messages or commit trailers.
- Keep commit messages clean and repository-focused.
- If a template, local git config, or copied message already contains a co-author or assistant-attribution line, remove it before finishing.
- If the user asks for a commit, return or create a normal commit message with no attribution footer.

## Examples

```text
feat(auth): tighten session validation
```

```text
fix(router): normalize nested layout file conventions
```

## Final Check

Before creating or suggesting any commit-related text, scan it once for:

- `Co-authored-by:`
- `Generated by`
- `Authored with`
- tool or model names in a footer

If any appear, remove them.
