This guide walks through deploying your first service.

## 1. Connect a repository

Point Otterdeploy at a Git repository. On the next push, the builder detects your
framework with Railpack and produces an image — no Dockerfile required.

## 2. Add a domain

Each service can have one or more domains. Otterdeploy provisions HTTPS through
the Caddy edge automatically and checks DNS reachability as you go.

## 3. Ship it

```bash
otterdeploy deploy
```

Watch the build and runtime logs stream live, and check per-resource metrics
from the resource panel.
