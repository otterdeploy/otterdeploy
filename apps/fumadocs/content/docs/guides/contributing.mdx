## Development Setup

Follow the [Installation](/docs/getting-started/installation) guide to set up your local environment.

## Project Structure

Otterdeploy is a Turborepo monorepo with Bun workspaces:

```
apps/
  web/           # React frontend
  server/        # Hono API server
  worker/        # Inngest job worker
  cli/           # otterdeploy CLI
  fumadocs/      # Documentation (this site)

packages/
  api/           # oRPC routers & procedures
  auth/          # Better Auth configuration
  contract/      # Shared Zod schemas
  db/            # Drizzle ORM schema
  domain/        # Business logic services
  env/           # Environment variable validation
  events/        # Inngest event definitions
  infra-config/  # Config loader, differ, reconciler
  logger/        # Pino logging
  secrets/       # Secrets management
  ui/            # Shared UI components
  zero/          # Zero reactive queries
```

## Development Commands

```bash
bun run dev              # Start all apps
bun run check-types      # TypeScript type checking
bun run check            # Oxlint + Oxfmt linting and formatting
bun run build            # Build all packages
```

## Making Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b feat/my-feature`
3. Make your changes
4. Run type checking: `bun run check-types`
5. Run linting: `bun run check`
6. Commit with a conventional commit message
7. Push and open a Pull Request

## Commit Convention

This project follows [Conventional Commits](https://www.conventionalcommits.org/):

| Prefix | Use for |
|---|---|
| `feat:` | New features |
| `fix:` | Bug fixes |
| `docs:` | Documentation changes |
| `refactor:` | Code refactoring |
| `chore:` | Build, CI, dependency updates |
| `test:` | Adding or updating tests |

Examples:

```
feat: add multi-domain support to infra-config
fix: resolve circular inheritance detection
docs: add YAML config guide
refactor(differ): extract domain diffing logic
```

## Database Changes

When modifying the Drizzle schema in `packages/db/src/schema/`:

1. Make your schema changes
2. Generate a migration: `bun run db:generate`
3. Apply it locally: `bun run db:push`
4. Test with Drizzle Studio: `bun run db:studio`

## Adding Documentation

Documentation lives in `apps/fumadocs/content/docs/`. Add `.mdx` files with frontmatter:

```mdx
---
title: My Page
description: What this page covers
---

Content here...
```

Update the relevant `meta.json` to include your page in the sidebar navigation.
