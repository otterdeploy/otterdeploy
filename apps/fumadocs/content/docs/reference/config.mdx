## File Formats

Otterdeploy auto-detects your config file by searching in this order:

1. `otterdeploy.yaml`
2. `otterdeploy.yml`
3. `otterdeploy.config.ts`
4. `otterdeploy.config.js`
5. `otterdeploy.config.mjs`

Use `--config <path>` to override auto-detection.

## Root Config

```typescript
{
  project: string,        // Project name (1-128 chars)
  organization: string,   // Organization ID
  environments: Record<string, EnvironmentConfig>,
}
```

## EnvironmentConfig

```typescript
{
  resources?: Record<string, ResourceConfig>,
  links?: LinkConfig[],
  extends?: string,                         // Parent environment name
  exclude?: string[],                       // Resources to remove from parent
  overrides?: {
    resources?: Record<string, ResourceOverride>,
  },
  extraLinks?: LinkConfig[],                // Links to append to inherited links
  removeLinks?: { from: string, to: string }[], // Links to remove from parent
}
```

## ResourceConfig

```typescript
{
  kind: "web" | "api" | "worker" | "database" | "cache" | "volume",

  source?: {
    github: string,       // "org/repo"
    branch?: string,      // default: "main"
    root?: string,        // default: "/"
  },

  image?: string,         // Container image (alternative to source + build)

  build?: {
    builder?: "nixpacks" | "dockerfile" | "buildpack" | "railpack",  // default: "nixpacks"
    buildCommand?: string,
    dockerfilePath?: string,
    watchPatterns?: string[],
  },

  deploy?: {
    startCommand?: string,
    preDeployCommand?: string,
    replicas?: number,                // default: 1, range: 1-200
    healthcheckPath?: string,
    healthcheckTimeout?: number,      // seconds
    restartPolicy?: "ON_FAILURE" | "ALWAYS" | "NEVER",
    restartPolicyMaxRetries?: number,
    cronSchedule?: string,            // cron expression
    region?: string,
    sleepApplication?: boolean,
    overlapSeconds?: number,          // old/new deploy overlap
    drainingSeconds?: number,         // SIGTERM to SIGKILL grace period
  },

  domain?: string | string[],        // one or more custom domains
  env?: Record<string, string>,       // environment variables
  storage?: string,                   // persistent storage size (e.g., "20Gi")
  server?: string,                    // target server ID
}
```

## ResourceOverride

All fields from `ResourceConfig` (all optional), plus:

```typescript
{
  excludeEnv?: string[],   // env var keys to remove from parent
}
```

## LinkConfig

```typescript
{
  from: string,            // source resource name
  to: string,              // target resource name
  type?: "depends_on" | "network" | "mounts",  // default: "depends_on"
}
```

## Reference Syntax

In `env` values, use `${resource.property}`:

| Reference | Resolves To |
|---|---|
| `${api.domain}` | Primary (first) domain of the `api` resource |
| `${db.connectionString}` | Auto-generated connection URI for `db` |

## TypeScript Helper

Use `defineConfig` for type checking and autocompletion:

```typescript
import { defineConfig } from "@otterdeploy/infra-config";

export default defineConfig({
  project: "my-app",
  organization: "org_abc123",
  environments: {
    production: {
      resources: { /* ... */ },
      links: [ /* ... */ ],
    },
  },
});
```
