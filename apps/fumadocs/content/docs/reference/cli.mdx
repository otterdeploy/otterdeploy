## Installation

The CLI is part of the `@otterdeploy/cli` package. In the monorepo, run it via:

```bash
bun run infra:init      # otterdeploy init
bun run infra:preview   # otterdeploy preview
bun run infra:sync      # otterdeploy sync
```

Or install globally:

```bash
bun add -g @otterdeploy/cli
```

## Commands

### `otterdeploy init`

Scaffold a new `otterdeploy.config.ts` template in the current directory.

```bash
otterdeploy init
```

Fails if a config file already exists.

### `otterdeploy preview`

Load the config, resolve inheritance and references, diff against current state, and print the changeset without applying it.

```bash
otterdeploy preview [options]
```

| Option | Description |
|---|---|
| `--config <path>` | Path to config file |
| `--env <name>` | Target a single environment |

### `otterdeploy sync`

Load the config, compute the diff, and apply all changes.

```bash
otterdeploy sync [options]
```

| Option | Description |
|---|---|
| `--config <path>` | Path to config file |
| `--env <name>` | Target a single environment |
| `--yes` | Skip confirmation prompt |
| `--preview` | Show changes before applying |

### `otterdeploy status`

Show current infrastructure state compared to the config file. Highlights drift.

```bash
otterdeploy status [options]
```

| Option | Description |
|---|---|
| `--config <path>` | Path to config file |
| `--env <name>` | Target a single environment |

### `otterdeploy destroy`

Generate and apply delete actions for all resources in the config.

```bash
otterdeploy destroy [options]
```

| Option | Description |
|---|---|
| `--config <path>` | Path to config file |
| `--env <name>` | Target a single environment |
| `--yes` | Skip confirmation prompt |

## Global Options

| Option | Description |
|---|---|
| `--help` | Show help |
| `--config <path>` | Override config file auto-detection |

## Config File Resolution

When `--config` is not provided, the CLI searches the current directory for:

1. `otterdeploy.yaml`
2. `otterdeploy.yml`
3. `otterdeploy.config.ts`
4. `otterdeploy.config.js`
5. `otterdeploy.config.mjs`

The first file found is used.
