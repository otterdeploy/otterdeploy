---
title: Inheritance
description: How environment inheritance and deep merging work
---

When an environment uses `extends`, Otterdeploy runs a **6-step inheritance algorithm** to produce the final resolved environment.

## The 6-Step Algorithm

### Step 1: Copy parent resources

Start with a copy of every resource from the parent environment.

### Step 2: Exclude resources

Remove any resources listed in `exclude`:

```yaml
staging:
  extends: production
  exclude: [redis, monitoring]
```

### Step 3: Deep-merge overrides

Apply `overrides.resources` with field-by-field merging:

```yaml
overrides:
  resources:
    api:
      source: { branch: develop }       # only branch changes, github + root kept
      build: { buildCommand: "npm test" } # only buildCommand changes, builder kept
      deploy: { replicas: 1 }            # only replicas changes, startCommand etc. kept
      env: { NODE_ENV: staging }          # merged key-by-key with parent env
      excludeEnv: [ANALYTICS]             # remove specific env vars
      domain: staging.api.myapp.com       # domain is fully replaced
```

**Merge rules by field:**

| Field | Merge Strategy |
|---|---|
| `source` | Field-by-field (branch override keeps github + root) |
| `build` | Field-by-field (buildCommand override keeps builder) |
| `deploy` | Field-by-field (replicas override keeps startCommand, healthcheck, etc.) |
| `env` | Key-by-key (new keys added, existing keys overwritten) |
| `excludeEnv` | Listed keys deleted from the merged env |
| `domain` | Full replacement (staging domains replace production domains) |
| `kind`, `image`, `storage`, `server` | Direct replacement |

### Step 4: Add child resources

Any resources defined directly in `resources` (not `overrides`) are added or replace inherited ones entirely:

```yaml
staging:
  extends: production
  resources:
    debug-api:        # new resource, only in staging
      kind: api
      deploy: { replicas: 1 }
```

### Step 5: Resolve links

- If the child defines `links` (non-empty array), those **replace all** inherited links
- Otherwise, inherit parent links, then:
  - Remove any matching `removeLinks`
  - Append any `extraLinks`

```yaml
staging:
  extends: production
  removeLinks:
    - { from: api, to: redis }       # remove this specific link
  extraLinks:
    - { from: web, to: db, type: network }  # add this link
```

### Step 6: Filter broken links

Any link referencing a resource that doesn't exist (e.g., excluded in step 2) is automatically removed.

## Full Example

```yaml
environments:
  production:
    resources:
      api:
        kind: api
        source: { github: org/app, branch: main }
        build: { builder: nixpacks, buildCommand: "npm run build" }
        deploy:
          startCommand: "node dist/index.js"
          replicas: 3
          healthcheckPath: /health
          region: us-west
        domain: [api.myapp.com, api-v2.myapp.com]
        env:
          NODE_ENV: production
          LOG_LEVEL: warn
          ANALYTICS: "true"
      web:
        kind: web
        deploy: { replicas: 2 }
        domain: [myapp.com, www.myapp.com]
      postgres:
        kind: database
        image: postgres:16
        storage: 20Gi
      redis:
        kind: cache
        image: redis:7-alpine
    links:
      - { from: api, to: postgres, type: depends_on }
      - { from: api, to: redis, type: depends_on }
      - { from: web, to: api, type: network }

  staging:
    extends: production
    exclude: [redis]
    overrides:
      resources:
        api:
          source: { branch: develop }
          deploy: { replicas: 1, region: eu-central }
          domain: api.staging.myapp.com
          env: { NODE_ENV: staging, LOG_LEVEL: debug }
          excludeEnv: [ANALYTICS]
        web:
          domain: staging.myapp.com
        postgres:
          storage: 5Gi
    extraLinks:
      - { from: web, to: postgres, type: network }

  dev:
    extends: staging
    overrides:
      resources:
        api:
          source: { branch: feature/dev }
          deploy: { replicas: 1, sleepApplication: true }
          env: { DEBUG: "*" }
```

### Resolved: staging

- **api** — branch=develop, replicas=1, region=eu-central, env: NODE_ENV=staging, LOG_LEVEL=debug (no ANALYTICS)
- **web** — domain=staging.myapp.com
- **postgres** — storage=5Gi
- **Links:** api->postgres, web->api, web->postgres (redis link auto-removed because redis was excluded)

### Resolved: dev

- **api** — branch=feature/dev, replicas=1, sleepApplication=true, env: NODE_ENV=staging, LOG_LEVEL=debug, DEBUG=\*
- **web** — same as staging
- **postgres** — same as staging
- **Links:** same as staging
