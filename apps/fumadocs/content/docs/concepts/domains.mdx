---
title: Domains
description: Assign custom domains to your resources
---

Otterdeploy supports assigning one or more custom domains to any resource.

## Single Domain

```yaml
api:
  kind: api
  domain: api.myapp.com
```

## Multiple Domains

Pass an array to assign multiple domains to one resource:

```yaml
api:
  kind: api
  domain:
    - api.myapp.com
    - api-v2.myapp.com
```

## Domain References

Use `${resource.domain}` in environment variables to reference a resource's primary domain (the first one in the array):

```yaml
web:
  kind: web
  env:
    API_URL: "https://${api.domain}"
```

If `api.domain` is `["api.myapp.com", "api-v2.myapp.com"]`, the resolved value will be `https://api.myapp.com`.

## Domain Inheritance

When overriding a resource's domain in a child environment, the domain is **fully replaced** (not merged):

```yaml
staging:
  extends: production
  overrides:
    resources:
      api:
        domain: api.staging.myapp.com   # replaces all production domains
```

## Diffing

During sync, Otterdeploy compares desired domains against current domains per resource and generates the minimal set of add/remove actions:

- New domains are created
- Removed domains are deleted
- Unchanged domains are left alone
