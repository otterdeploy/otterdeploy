## Design Context

The frontend design system is documented in two root files — read them before building or changing any UI:

- **`PRODUCT.md`** (strategic): register is **product**; users are indie devs + platform engineers; personality is **"calm, confident infrastructure."** Five principles: tool-disappears-into-task, honest-about-system-state, calm-density, fast-is-a-feature, one-coherent-vocabulary. Anti-references: cluttered self-host UIs, enterprise cloud consoles, generic AI-SaaS templates.
- **`DESIGN.md`** (visual): North Star **"The Quiet Instrument."** Warm-neutral monochrome (ink `#141412`, canvas `#fbfbfa`) + one OKLCH blue accent (≤10% of any screen). One type family — Geist sans (human) + Geist mono (machine: IDs/hashes/logs). Flat by default — hairline `ring-1 ring-foreground/10`, shadows only on floating layers. Tokens live in `apps/web/src/index.css`. The `.impeccable/` dir holds the machine-readable sidecar + live-mode config.

These were generated by the `impeccable` design skill (`/impeccable`). WCAG 2.2 AA in both light and dark; honor `prefers-reduced-motion`. Never ship fake/seeded data — unbuilt features get a "Coming soon" placeholder.
